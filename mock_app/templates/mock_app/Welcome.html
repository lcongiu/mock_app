Welcome!

This is a template app. Here are covered some basics of oTree.

First of all, oTree is based on the Django framework (official documentation here).
Therefore, any oTree App will be based the typical Django project files.


Specifically, any App will contain:

  • A "models.py" file, where code for the main classes (Constants, Session, Player, Groups) is written; unlike the typical Django App, code for Pages is contained in "pages.py".
  • Various "templates" files, that are the html pages to be shown on the browser; html pages/templates are contained in the folder "name_app/templates".

Any App is contained in a project. Among the project folders and files, we find:

  • A "urls.py" file, that in a Django project specifies the addresses of all resources needed for the project (in oTree, this will be managed automatically, thus no intervention is needed).
  • A "settings.py" file, where each App of the project must be manually listed, together with its settings.
  • A "_static" and "_templates" folder, where global templates (html) and styles (css) can be stored and retrieved once an App page is created.
