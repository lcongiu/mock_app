Thanks to the integration with Bootstrap, any element of our oTree App is responsive, i.e. its width scales with screen width.

However, there are two cases where the content may not be responsive:

  • When minimum widths are specified
  • When using particularly large tables

Minimum Widths

The first case in which content may be not responsive is when a minimum width is specified (in absolute terms). For example, this paragraph was put into a container whose minimum width has been set to 700px ("min-width: 700px"). If you try to resize the window below 700px, you will see that part of this paragraph will flow outside the window.

Using minimum widths per se is not a problem if we can be confident that the window will not be resized under a certain threshold (e.g., the task is performed under the supervision of the experimenter). But in general, it may be wise either to use RELATIVE min-width (%) or not specify any min width altogether (recall how the grid system works, as shown at page Bootstrap of this tutorial).


Large tables

The second case in which responsiveness is not guaranteed is when the page displays a particularly large table. Below, we included an example of such a table. Its properties have been specified in the style block of this page (check html code for details). In particular, we used the property "overflow-x:scroll". This property generates a scroll bar to scroll table content horizontally. Mind that the scroll bar will be visible only when the table width exceeds that of the screen.
